# 08 · 확장 아이디어

[[00-프로젝트-계획서|← 계획서로 돌아가기]]

> 코어를 완성한 뒤(5주차 이후) 원하는 방향으로 키우는 옵션들.
> 하나만 골라도 충분하다.

---

## A. 머신러닝 악성 앱 분류기 🤖

정상/악성 APK에서 특징(권한·API·문자열)을 뽑아 분류 모델 학습.

- 필요: `scikit-learn`, 라벨링된 데이터셋
- 결과물: "이 앱이 악성일 확률 87%" 같은 예측
- 어필: 보안 + ML 융합 → 이력서 차별화
- 난이도: 중상 (데이터셋 확보가 관건)

## B. 웹 대시보드 🌐

APK를 업로드하면 브라우저에서 리포트를 보는 UI.

- 필요: Flask/FastAPI + 간단한 프론트
- 결과물: 데모하기 좋은 시각적 산출물
- 어필: 면접에서 바로 보여줄 수 있음
- 난이도: 중

## C. 동적 분석 (Frida 후킹) 🔬

앱을 실제로 실행하면서 네트워크·파일·API 호출을 가로채 관찰.

- 필요: 안드로이드 기기/에뮬레이터, Frida
- 결과물: 정적 분석이 못 보는 런타임 행위 포착
- 어필: **N-day 분석(방향 A)으로 가는 다리**
- 난이도: 상 → 여기서 재미 붙으면 취약점 리서치로 진로 확장

## D. OWASP Mobile Top 10 매핑 📋

각 탐지 룰을 OWASP Mobile Top 10 항목에 매핑.

- 필요: 문서 작업 위주
- 결과물: "이 도구는 M1, M3, M9를 커버한다" → 자격증·실무 설득력
- 난이도: 하 (가성비 최고)

## E. 성능 최적화 ⚡

딥 분석 38초 → 단축.

- 문자열을 전부 훑지 말고 관심 패턴만 스트리밍 검사
- 멀티프로세싱으로 여러 APK 병렬 스캔
- 난이도: 중

## 추천 순서 (포트폴리오 관점)

1. **D (OWASP 매핑)** — 가성비 최고, 먼저
2. **B (웹 대시보드)** — 데모용
3. **A (ML 분류)** 또는 **C (동적 분석)** — 진로에 맞춰 택1
