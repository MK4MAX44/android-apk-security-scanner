## 1. 문제: 안드로이드는 왜 위험한가

안드로이드는 **개방형** 생태계다. 애플과 달리 공식 스토어 밖에서도 APK를
직접 설치(사이드로딩)할 수 있어, 악성코드가 들어올 통로가 구조적으로 넓다.
실제로 모바일 악성코드의 대부분이 안드로이드를 노린다.

그래서 **"이 APK가 안전한가?"** 를 빠르게 판별하는 능력은
보안 실무에서 계속 쓰이는 기본기다. 이 프로젝트는 그 판별을 자동화한다.

---

## 2. 이 프로젝트가 왜 좋은가

### ✅ 결과물이 확실하게 남는다
연구형 프로젝트(제로데이 찾기)와 달리, 이건 **반드시 동작하는 도구**가 나온다.
"운이 나빠서 아무것도 못 만들었다"가 없다. → [[02-목표와-범위]]

### ✅ 보안의 핵심 개념을 손으로 익힌다
권한 모델, 매니페스트, 앱 서명, 코드 난독화, 공격면(attack surface) 같은
안드로이드 보안의 기둥 개념을 **글로 읽는 게 아니라 코드로 다룬다.**

### ✅ 시작 문턱이 낮고 확장 천장이 높다
파이썬만 알면 오늘 시작할 수 있는데, 원하면 머신러닝 분류·동적 분석·웹
대시보드까지 계속 키울 수 있다. → [[08-확장-아이디어]]

### ✅ 돈이 안 든다
전부 오픈소스. 현금 예산 0원. → [[04-예산과-리소스]]

## 3. 나에게 구체적으로 어디에 도움이 되는가

### 🎯 포트폴리오 / 이력서
- GitHub에 **"직접 만든 보안 도구"** 가 올라간다. 채용에서 가장 강력한 한 줄.
- README + 분석 리포트 예시 = 면접에서 바로 보여줄 데모.
- "androguard로 APK 정적 분석 파이프라인을 설계했다"는 문장이 이력서에 들어간다.

### 🎯 취업 (Blue Team / 앱 보안)
- 모바일 앱 보안 진단, 악성 앱 분석은 **실무 직무**다.
- 이 도구가 하는 일(권한 분석, 매니페스트 점검, 시크릿 탐지)은
  실제 상용 도구(MobSF 등)의 축소판이다 → 상용 도구를 "쓸 줄만" 아는 지원자와 차별화.

### 🎯 자격증 대비
- 모바일 해킹/보안 자격증에서 다루는 개념(APK 구조, 권한, 서명, OWASP Mobile)을
  실습으로 커버한다.
- OWASP Mobile Top 10 항목 중 여러 개를 이 도구로 직접 탐지해 볼 수 있다.

### 🎯 실력 그 자체
- 리버싱·정적 분석의 **입구**. 여기서 재미를 붙이면
  [[08-확장-아이디어]]의 동적 분석(Frida) → N-day 분석으로 자연스럽게 넘어간다.

## 4. 왜 "지금" 이 방식인가

| 대안 | 문제 |
|------|------|
| 제로데이 직접 찾기 | 몇 달~몇 년, 성과 불확실 → 초심자에겐 좌절 |
| 상용 도구(MobSF)만 사용 | 남이 만든 걸 쓸 뿐, 원리를 모름 |
| **직접 축소판 제작 (이 프로젝트)** | **원리를 이해하면서 결과물도 남음** ← 최적 |

---

## 5. 완성했을 때 말할 수 있는 것

> "안드로이드 APK를 정적 분석해서 위험 권한, 잘못된 매니페스트 설정,
> 하드코딩된 비밀키를 자동 탐지하고 심각도 점수를 매기는 도구를
> androguard 기반으로 직접 설계·구현했습니다. 실제 오픈소스 앱과
> 악성 샘플로 검증했습니다."

이 한 문단이 이 프로젝트의 최종 목표다.
