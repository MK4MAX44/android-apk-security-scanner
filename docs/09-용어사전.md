# 09 · 용어사전

[[00-프로젝트-계획서|← 계획서로 돌아가기]]

> 모르는 단어가 나오면 여기서 찾기.

---

**APK** (Android Package)
안드로이드 앱 설치 파일. 실제로는 ZIP 압축 파일이라, 열어보면 매니페스트·코드·리소스가 들어있다.

**AndroidManifest.xml (매니페스트)**
앱의 신분증. 패키지명, 요청 권한, 컴포넌트 목록, 보안 설정이 다 여기 있다.
단, APK 안에서는 사람이 못 읽는 바이너리(AXML)로 압축돼 있어 androguard가 풀어준다.

**DEX** (Dalvik Executable)
안드로이드 앱의 실제 실행 코드(자바/코틀린을 컴파일한 것). `classes.dex`.
여기서 문자열·API 호출을 뽑아 분석한다.

**권한(Permission)**
앱이 카메라·위치·SMS 같은 민감 기능을 쓰려면 선언해야 하는 것.
구글이 위험도로 normal / dangerous 등급을 나눈다. → [[07-탐지-룰-설명]]

**exported (노출) 컴포넌트**
다른 앱이 호출할 수 있게 열린 activity/service/receiver/provider.
공격자가 접근할 수 있는 **공격면(attack surface)** 이 된다.

**ContentProvider**
앱 간 데이터 공유 통로. exported면 다른 앱이 데이터를 읽을 수 있어 주의.

**서명(Signing) / 인증서**
모든 APK는 개발자 키로 서명된다. 같은 키로 서명돼야 앱 업데이트가 된다.
**디버그 키**는 모두가 공유하는 공개 키라 릴리스에 쓰면 위험. → CERT-001

**정적 분석 (Static Analysis)**
앱을 **실행하지 않고** 파일만 뜯어보는 분석. 이 프로젝트의 방식. 안전하다.

**동적 분석 (Dynamic Analysis)**
앱을 **실제로 실행**하면서 행위를 관찰. Frida 등 사용. → [[08-확장-아이디어]]

**하드코딩된 시크릿**
API 키·토큰·비밀번호를 코드에 그대로 박아 넣은 것. 디컴파일하면 다 보인다. → SECRET-*

**난독화 (Obfuscation)**
코드를 일부러 알아보기 어렵게 만드는 것. 정적 분석을 방해한다.

**androguard**
APK를 파싱·분석하는 파이썬 라이브러리. 이 도구의 엔진. → [[05-아키텍처]]

**공격면 (Attack Surface)**
공격자가 건드릴 수 있는 모든 지점의 총합. 적을수록 안전.

**심각도 (Severity)**
위험의 정도. INFO < LOW < MEDIUM < HIGH < CRITICAL.

**오탐 / 미탐 (False Positive / False Negative)**
오탐 = 멀쩡한데 위험하다고 잘못 표시. 미탐 = 위험한데 못 잡음.
둘의 균형이 탐지 도구의 품질. → [[03-일정-로드맵]] 4주차

**OWASP Mobile Top 10**
모바일 앱의 대표 보안 위험 10가지 목록. 업계 표준 체크리스트. → [[08-확장-아이디어]]

**N-day / 제로데이(0-day)**
0-day = 아직 안 알려진(패치 없는) 취약점. N-day = 이미 공개·패치된 취약점.
우리가 처음 고민했던 방향 A가 N-day 분석. → [[02-목표와-범위]]
