## 설치 (최초 1회)

```bash
cd ~/apkscan
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

---

## 기본 실행

```bash
# 사람이 읽는 컬러 리포트 (딥 분석: 코드까지)
.venv/bin/python -m apkscan samples/app.apk

# 빠른 스캔 (매니페스트·인증서만, ~1초)
.venv/bin/python -m apkscan samples/app.apk --fast
```

---

## 리포트 형식 선택

```bash
# JSON (다른 도구·스크립트 연동용)
.venv/bin/python -m apkscan app.apk --json -o reports/app.json

# 마크다운 (옵시디언에서 보기 좋음)
.venv/bin/python -m apkscan app.apk --markdown -o reports/app.md

# HTML 대시보드 (브라우저에서 바로 열림, 서버 불필요)
.venv/bin/python -m apkscan app.apk --html -o reports/app.html
open reports/app.html    # macOS

# 색상 끄기 (로그 파일로 저장 시)
.venv/bin/python -m apkscan app.apk --no-color
```

---

## 옵션 정리

| 옵션 | 설명 |
|------|------|
| `--fast` | DEX 코드 분석 생략 (빠름) |
| `--json` | JSON 출력 |
| `--markdown` / `--md` | 마크다운 출력 |
| `--html` | HTML 대시보드 출력 (브라우저용) |
| `-o PATH` | 파일로 저장 |
| `--no-color` | 터미널 색상 끄기 |

환경변수 `APKSCAN_VERBOSE=1` → androguard 상세 로그 켜기 (디버깅용).

## 종료 코드 (CI 연동 대비)

| 코드 | 의미 |
|:---:|------|
| 0 | 위험 점수 < 50 (양호) |
| 3 | 위험 점수 ≥ 50 (주의) |
| 1 | 분석 실패 |
| 2 | 파일 없음 |

## 실전 예: 검증에 썼던 명령

```bash
# F-Droid 오픈소스 앱으로 동작 검증
.venv/bin/python -m apkscan samples/fdroid-basic.apk --markdown \
    -o reports/fdroid-basic.md
```

→ 결과 리포트는 `reports/` 폴더에서 확인. (예시가 이미 생성돼 있음)

## 분석용 APK는 어디서?

- **정상 앱:** [F-Droid](https://f-droid.org) 오픈소스 APK (합법·안전)
- **취약 실습 앱:** 교육용 취약 앱 프로젝트 (일부러 취약하게 만든 학습용)
- ⚠️ 악성 샘플은 **격리 환경에서 정적 분석만**. 절대 실행하지 말 것. → [[04-예산과-리소스]]
