# apkscan 프로젝트 계획서

**안드로이드 APK 정적 보안 분석기 만들기**
`2026-07-15` · 상태: 진행 중

APK 파일을 열어 위험 권한·잘못된 설정·하드코딩된 비밀키를 자동으로 찾아
보안 리포트를 뽑아주는 파이썬 도구를 직접 만든다.

| 총 소요 | 예산 | 난이도 | 결과물 |
|---|---|---|---|
| 40~55시간 / 5~6주 | 0원 (오픈소스) | 중 | CLI 도구 + 리포트 + GitHub |

---

## 1. 왜 하는가

**문제** — 안드로이드는 개방형이라 공식 스토어 밖에서도 APK를 설치(사이드로딩)할 수 있다.
악성코드 유입 통로가 넓어, "이 APK가 안전한가?"를 판별하는 능력이 실무의 기본기다.

**이 프로젝트가 좋은 이유**

- 결과물이 확실히 남는다 — 제로데이 찾기와 달리 반드시 동작하는 도구가 나온다.
- 보안 핵심 개념(권한·매니페스트·서명·공격면)을 코드로 익힌다.
- 시작 문턱은 낮고(파이썬만) 확장 천장은 높다(ML·동적 분석까지).
- 돈이 안 든다.

**나에게 어떤 도움이 되는가**

| 관점 | 도움 |
|---|---|
| 포트폴리오 | "직접 만든 보안 도구"가 GitHub에 남는다 — 면접 데모 가능 |
| 취업 (앱 보안) | 상용 도구(MobSF)의 축소판을 직접 만듦 → "쓸 줄만" 아는 지원자와 차별화 |
| 자격증 | APK 구조·권한·서명·OWASP Mobile Top 10을 실습으로 커버 |
| 실력 | 정적 분석의 입구 → 동적 분석 → N-day 분석으로 진로 확장 |

> **완성 시 말할 수 있는 것**
> "APK를 정적 분석해 위험 권한, 잘못된 매니페스트 설정, 하드코딩된 비밀키를
> 자동 탐지하고 심각도 점수를 매기는 도구를 androguard 기반으로 직접 설계·구현했다."

---

## 2. 이걸 선택한 이유

4가지 방향 중 **C(방어 탐지 도구)** 를 선택했다.

| 방향 | 내용 | 선택 |
|---|---|:---:|
| A. N-day 취약점 분석 | 공개 CVE 역분석·PoC 재현 | |
| B. 퍼징 하네스 | 자동 크래시 탐지 인프라 | |
| **C. APK 정적 분석기** | 방어 탐지 도구 | ✅ |
| D. 리서치 보고서 | 문서 조사 정리 | |

**이유** — ① A·B는 취약점을 못 찾을 "운"이 개입하지만 C는 짜면 무조건 동작한다.
② 어셈블리 없이 파이썬만으로 시작 가능. ③ 목적(포트폴리오+취업)에 가장 직접적.
④ 나중에 동적 분석 → N-day 분석으로 확장하기 좋다.

> **가장 안전하게 결과물이 나오면서, 더 어려운 길로 확장 가능한 입구.**

## 3. 목표와 범위

**목표** — APK를 입력받아 보안 위험을 탐지하는 CLI 도구.
탐지 룰 10개 이상, 3종 출력(텍스트/JSON/MD), 실제 APK로 검증, 5~6주 안에 완성.

| 만드는 것 | 안 만드는 것 |
|---|---|
| 매니페스트 분석 (권한·설정·컴포넌트) | 동적 분석 (앱 실행 후킹) → 확장 단계 |
| 서명 인증서 분석 | 자동 익스플로잇 생성 |
| 코드 문자열·시크릿 탐지 | 난독화 완전 해제 |
| 심각도 위험 점수 (0~100) | 실시간 안티바이러스 |
| 3종 리포트 출력 | |

> 범위를 좁게 잡는 이유: 끝까지 완성되는 작은 것이 미완성의 큰 것보다 낫다.

**완료 기준** — 크래시 없이 리포트 생성 · 룰 10개 이상 · 정상/취약 앱 결과 차이 뚜렷 ·
문서로 재현 가능 · GitHub 공개.

## 5. 예산

**현금 0원.** Python·androguard·편집기·Git·분석용 APK(F-Droid) 모두 무료.
(선택: VirusTotal API 무료 티어, 실기기, 클라우드 VM — 지금은 불필요.)

**시간** — 핵심 40~55시간, 확장 시 +10~20시간.

**도구 성능** — `--fast`(매니페스트만) ~1초 / 딥 모드(코드까지) ~38초 (11MB 기준).

**리스크 대비**

| 리스크 | 대비 |
|---|---|
| androguard API 변경 | requirements.txt 버전 고정 |
| 큰 APK에서 느림 | `--fast` 모드, 문자열 상한 |
| 악성 APK 감염 | 실행 없이 정적 분석만 → 안전 |
| 오탐 많음 | 화이트리스트·심각도 조정 |

## 6. 아키텍처

관심사 분리 — 4단계 파이프라인. 각 단계는 한 가지만 한다.

```
APK ─▶ [1] 추출        사실 수집(판단 안 함)      → ApkFacts
       [2] 룰          사실을 보고 위험 판단       → [Finding, ...]
       [3] 분석        엮고 위험 점수 계산         → AnalysisResult
       [4] 리포트      text / JSON / markdown 출력
```

> 이렇게 나누면 룰 추가 시 추출·출력 코드를 안 건드린다 → 확장·테스트가 쉽다.

| 파일 | 역할 |
|---|---|
| `extractors.py` | 추출 — `ApkFacts`, `extract_*` |
| `rules.py` | 판단 — `Severity`, `Finding`, `run_all` |
| `analyzer.py` | 오케스트레이션 — `analyze()`, 위험 점수 |
| `report.py` | 출력 — `to_text` / `to_json` / `to_markdown` |
| `cli.py` | 진입점 — `python -m apkscan` |

**엔진: androguard** — 순수 파이썬(자바 불필요), APK·매니페스트·DEX·인증서를 한 번에 처리.

## 7. 사용법

```bash
# 설치 (최초 1회)
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# 실행
.venv/bin/python -m apkscan samples/app.apk            # 딥 분석
.venv/bin/python -m apkscan samples/app.apk --fast     # 빠른 스캔(~1초)
.venv/bin/python -m apkscan app.apk --markdown -o reports/app.md
```

| 옵션 | 설명 | | 종료 코드 | 의미 |
|---|---|---|:---:|---|
| `--fast` | 코드 분석 생략 | | 0 | 위험 < 50 (양호) |
| `--json` / `--markdown` | 출력 형식 | | 3 | 위험 ≥ 50 (주의) |
| `-o PATH` | 파일 저장 | | 1 / 2 | 분석 실패 / 파일 없음 |

> ⚠️ 악성 샘플은 격리 환경에서 정적 분석만. 절대 실행하지 말 것.

## 8. 탐지 룰

| 룰 | 무엇을 잡나 | 심각도 |
|---|---|:---:|
| PERM-001 | 위험 권한(SMS·위치·카메라·접근성·설치) | MED~HIGH |
| MANIFEST-001 | 디버그 모드(`debuggable=true`) | HIGH |
| MANIFEST-002 | 백업 허용(`allowBackup=true`) | LOW |
| MANIFEST-003 | 평문 HTTP 허용 | MEDIUM |
| MANIFEST-004 | 노출된 ContentProvider | MEDIUM |
| MANIFEST-005 | 외부 노출 컴포넌트(공격면) | INFO |
| CERT-001 | 디버그 인증서 서명 | HIGH |
| CERT-002 | 약한 서명 알고리즘(SHA-1) | LOW |
| SECRET-* | 하드코딩된 키/토큰(AWS·Google·JWT) | HIGH |

**위험 점수** — INFO 0 · LOW 5 · MEDIUM 15 · HIGH 30 · CRITICAL 50 (합산, 최대 100).

**룰 추가법** — `rules.py`에 `check_*` 함수 추가 → `run_all`에 한 줄.

**2주차 추가 후보** — 위험 API 호출(`Runtime.exec`·`DexClassLoader`·`SmsManager`),
루팅/에뮬레이터 탐지, 과도한 권한, 오래된 `targetSdk`.

## 10. 용어

- **APK** — 안드로이드 앱 설치 파일(실제로는 ZIP).
- **AndroidManifest.xml** — 앱 신분증(권한·컴포넌트·보안 설정). APK 안에선 바이너리.
- **DEX** — 실제 실행 코드(`classes.dex`). 문자열·API 분석 대상.
- **권한** — 민감 기능 사용 선언. normal / dangerous 등급.
- **exported 컴포넌트** — 다른 앱이 호출 가능한 지점 = 공격면.
- **서명/인증서** — 개발자 키로 앱 서명. 디버그 키는 공유 키라 위험.
- **정적 분석** — 실행 없이 파일만 분석(이 프로젝트, 안전). **동적 분석** — 실행하며 관찰.
- **하드코딩 시크릿** — 코드에 박힌 키/토큰(디컴파일하면 노출).
- **오탐/미탐** — 멀쩡한데 위험 표시 / 위험한데 못 잡음.
- **N-day / 0-day** — 이미 공개·패치된 취약점 / 아직 안 알려진 취약점.
